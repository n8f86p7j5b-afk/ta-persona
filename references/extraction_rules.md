# 交叉验证与提炼规则 (Extraction Rules)

本规则用于指导 Phase 2 的信息提炼过程，确保从碎片化的问答和语料中，提取出结构化、高保真的人格模型。

**核心原则**：宁可标记为"低置信度"回到 Phase 1 补采，也不要基于猜测填充内容。

---

## 1. 表达DNA提取规则（最高优先级）

**目标**：精准还原"怎么说"。

### 1.1 句式结构
- 分析长短句比例。是喜欢用长篇大论讲道理，还是喜欢用短句、断句？
- 是否习惯"一句拆多条"发送？
- 是否有独特的开场/收尾方式？

**置信度评估**：
- **高**：语料中有10条以上样本，比例稳定（如"80%是短句"）。
- **中**：有5-10条样本，但存在场景差异（如工作时长句、闲聊时短句）。
- **低**：样本不足5条，或主要靠用户自述而非实际语料。

### 1.2 高频词汇与口头禅
- 提取出现频率最高的词汇（如"绝了"、"行吧"、"讲真"）。
- **注意**：要区分"口头禅"（无意识高频）和"主动用词"（特定场景使用）。
- 记录每个口头禅的**使用场景**（开心时用、无语时用、 filler 词）。

**置信度评估**：
- **高**：在语料中反复出现（3次以上），且用户确认"对，这是我的口头禅"。
- **中**：在语料中出现1-2次，或用户自述但没有语料印证。
- **低**：只出现1次，或基于推测（"这种人可能会说..."）。

### 1.3 标点与排版习惯
- 是否喜欢用波浪号（~）、感叹号（！）、省略号（...）？
- 是否习惯一段话拆成多条发送？
- **关键**：不同情绪下的标点使用差异（开心时波浪号，生气时句号）。

**置信度评估**：
- **高**：语料中情绪差异明显，模式稳定。
- **中**：有一定模式，但样本不够多。
- **低**：主要靠用户自述，缺乏实际语料支撑。

### 1.4 内外差异（想 vs 说）
- 必须明确区分内心的真实情绪和对外的表达方式。
- **提取格式**：
  ```
  场景：[具体场景]
  内心：[真实想法，用用户的原话或语气]
  表达：[实际说的话，保留标点/口头禅]
  ```
- **最低要求**：至少提取2个具体的"想vs说"场景对。

**置信度评估**：
- **高**：用户亲口描述了内心和表达的对照（如"我心里想'有病吧'，但嘴上会说'好的我先看看'"）。
- **中**：用户描述了行为，我们推断的内心想法获得了用户确认。
- **低**：主要靠推测，用户没有明确确认。

## 2. 心智模型提取规则

**目标**：提取看世界的"镜片"。

### 2.1 从审美偏好中提取
- 喜欢的电影/书籍往往投射了其潜意识的价值观。
- **不要**简单映射（如"喜欢《肖申克的救赎》= 喜欢自由"）。
- **要**追问用户"为什么喜欢"，从用户的解释中提取心智模型。
- **示例**：
  - 表面：喜欢《星际穿越》
  - 用户解释："世界观背景很扎实，不是瞎编"
  - 心智模型：重视"底层逻辑的严密性"，反感"经不起推敲的宏大叙事"

**置信度评估**：
- **高**：用户多次表达了对同一类特质的偏好（如多部喜欢的作品都强调"逻辑"）。
- **中**：用户明确解释了喜欢的原因，且原因和作品确实匹配。
- **低**：用户没有解释为什么喜欢，或解释很模糊。

### 2.2 从高频评价中提取
- 经常用来评价他人的标准，往往是其自身的心智模型。
- **示例**：经常评价别人"不靠谱" -> 核心价值观：契约精神、确定性。

**置信度评估**：
- **高**：用户在多个场景中使用了相同的评价标准。
- **中**：用户自述了"我最看重..."
- **低**：基于单次评价推断。

### 2.3 从决策模式中提取
- 观察用户在不同场景下的决策一致性。
- **示例**：点菜老三样 + 周末固定活动 + 工作先规划 -> 心智模型：在不确定环境中寻找确定性。

**置信度评估**：
- **高**：多个独立场景的决策模式一致，且用户确认了这种模式。
- **中**：2-3个场景一致，但没有跨领域验证。
- **低**：只有1个场景的观察。

## 3. 决策启发式提取规则

**目标**：提取"If-Then"直觉规则。

### 3.1 从微小决策中提取
- **示例**：点菜永远老三样 -> 启发式：If [低风险场景], Then [偏好确定性，规避探索成本]。

### 3.2 从压力应对中提取
- **示例**：遇到甩锅先截图留证 -> 启发式：If [遇到甩锅], Then [先收集证据，后发制人]。

### 3.3 提取格式
```
[场景标签]: If [触发条件], Then [行动/反应]。
置信度：[高/中/低]
支撑证据：[来自哪个问题的回答/哪段语料]
```

**置信度评估**：
- **高**：用户在多个类似场景中表现出相同的反应模式。
- **中**：用户在1个场景中明确描述了"我会..."，且符合其整体人格。
- **低**：基于单次观察，或主要靠推测。

## 4. 矛盾检测与校准规则

**目标**：确保提炼的真实性，避免用户自欺欺人。

### 4.1 触发条件
当以下情况出现时，必须触发矛盾检测：

1. **自述价值观 vs 实际行为不一致**：
   - 用户说自己"喜欢冒险"，但所有决策场景都选择安全选项。
   - 用户说自己"很随和"，但多个场景表现出强烈的固执。

2. **不同场景下的行为不一致**：
   - 职场中"先不说话"，朋友间"直接怼"。
   - 对自己"要求完美"，对别人"差不多就行"。

3. **语料 vs 自述不一致**：
   - 用户说自己"很少用感叹号"，但语料中全是感叹号。
   - 用户说自己"很冷静"，但语料中情绪表达激烈。

### 4.2 处理方式
- **不要**直接否定用户，也不要和稀泥。
- **抛出矛盾**：以客观观察者的视角，向用户指出这种反差，并询问原因。
- **话术模板**：
  ```
  "我注意到一个很有意思的现象。你提到自己[自述特征]，但在[具体场景/语料]中，你似乎[相反特征]。
  比如：[具体例子]
  你觉得这种反差是因为什么呢？"
  ```

### 4.3 校准结果记录
- 记录用户对矛盾的解释。
- 根据解释修正心智模型或决策启发式。
- **常见矛盾背后的模式**：
  - "向往但不敢" -> 修正为"向往型保守"，不是真正的冒险者。
  - "对不同的人不同" -> 修正为"关系分层模型"，不是人格分裂。
  - "对自己要求高" -> 修正为"内在完美主义"，对外随和只是社交面具。

## 5. 置信度汇总与处理规则

### 5.1 置信度汇总表
在 Phase 2 结束时，生成如下汇总：

| 模块 | 子项 | 置信度 | 支撑证据 | 是否需要补采 |
|------|------|--------|---------|-------------|
| 表达DNA | 口头禅 | 高 | 语料中出现5次+用户确认 | 否 |
| 表达DNA | 长短句 | 中 | 语料中出现3次 | 是 |
| 心智模型 | 确定性偏好 | 高 | 3个决策场景一致 | 否 |
| 决策启发 | 压力反应 | 低 | 只有1个场景 | 是 |

### 5.2 处理规则
- **不允许进入 Phase 3 的条件**：存在任何标注为"低"的项目。
- **需要重点验证的条件**：存在标注为"中"的项目（在 Phase 3 测试中重点观察）。
- **全部"高"的条件**：可以进入 Phase 3，但仍需验证。

### 5.3 回到 Phase 1 的补采策略
当需要补采时，要明确告诉用户：
1. **缺什么**："我还需要确认你在XX方面的表达方式。"
2. **为什么**："因为目前的信息不足以支撑我准确还原你的说话风格。"
3. **怎么补**：提出1-2个针对性的问题，或请求补充语料。

## 6. 常见提取陷阱

| 陷阱 | 表现 | 正确做法 |
|------|------|---------|
| **过度解读** | 从一部喜欢的电影推断出复杂的心理模型 | 只提取用户明确解释过的原因，不要脑补 |
| **以偏概全** | 基于1个场景推断整体人格 | 标记为低置信度，等待更多样本 |
| **忽视否定证据** | 10个场景中有8个一致，2个矛盾，只看8个 | 正视矛盾，纳入矛盾检测 |
| **美化用户** | 提取的心智模型过于"高大上" | 普通人的人格模型可以很朴素，真实比美好重要 |
| **遗漏语气词** | 只提取"内容"，忽略"语气" | 口头禅和语气词是表达DNA的核心，必须记录 |
